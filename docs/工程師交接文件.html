六、權限系統架構

三層權限

層級實作方式程式碼位置
JWT 認證authenticate(req,res,next)middleware/auth.js
角色授權authorize('boss','manager')middleware/auth.js
功能權限requirePermission('finance','read')middleware/auth.js

CRUD + Scope

每個權限不只是有/沒有，而是分成 4 個操作 + 3 個範圍：

CRUDScope
can_create · can_read · can_update · can_deleteown=只看自己 · store=看本店 · all=看全部

使用 applyScopeFilter(query, req, userIdCol, storeIdCol) 自動加 WHERE 條件。

八、本機開發怎麼跑

# 1. 啟動資料庫
cd tongbao
docker compose up -d db redis

# 2. 安裝依賴
cd backend && npm install
cd ../frontend && npm install

# 3. 設定環境變數
cp .env.example .env
# 編輯 .env（DB密碼改成 tongbao_dev_2026）

# 4. 執行種子資料
cd backend && npx knex seed:run

# 5. 啟動後端
node src/app.js
# → http://localhost:3000

# 6. 啟動前端（另一個終端）
cd frontend && npx vite
# → http://localhost:5173

# 7. 登入
# 帳號: boss / staff / worker / manager / finance / client
# 密碼: 1234
